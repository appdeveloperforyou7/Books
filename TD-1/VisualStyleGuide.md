# THE GLITCH SQUAD — Visual Style Guide (Geronimo Edition)

> Think *Geronimo Stilton* with circuits. Every page should feel like a kid's desk after a creative explosion — messy, alive, full of personality, with COLORED WORDS jumping off the page, SOUND EFFECTS bursting through text, and illustrations woven into the reading experience.

**Companion file:** `VoiceGuide.md` — covers character voice, humor, and AI writing prompts. This file covers DESIGN + GERONIMO TYPOGRAPHY.

---

## 1. The Feel

**If this book were a place, it would be:**
A treehouse where four kids from different worlds built something together. One wall has Maya's blueprints taped up with circuits drawn in marker. Another has Zara's murals and collages. The floor is covered in Sam's game cartridges and Leo's printouts. Blip sits on a shelf, watching everything with his little LED face. It's controlled chaos — every piece has a reason, but nothing matches, and that's the magic.

**It is NOT:**
- A textbook
- A clean corporate design
- A minimalist Scandinavian layout
- Uniform or symmetrical
- Boring. Ever.

## 2. Page Types

Every page falls into one of these types. The variety between them is what keeps kids turning pages.

### TYPE A: Story Text Page
- 150-300 words of story text
- Body text in a warm, readable font
- At minimum: one Blip marginalia (tiny LED face in the margin reacting to the scene)
- Character-themed decorative elements in corners (see Section 4)
- Text wraps around occasional spot illustrations
- Colored pull-quotes for key lines or catchphrases

### TYPE B: Half-Illustration Page
- Large illustration occupying 40-60% of the page
- Story text flows around or below the image
- Image is NOT in a rectangle — see Section 3 for framing
- The illustration's background extends into the page margin slightly (bleed effect)
- Small caption or dialogue bubble overlay on the image

### TYPE C: Full-Page Illustration
- Full bleed illustration, no margins
- Maximum one line of text: a chapter title, a catchphrase, or Blip's reaction
- Used for: action climaxes, emotional peaks, location reveals, character introductions
- The image IS the page. No border. No frame. Just story.

### TYPE D: Action Sequence Page
- **Default**: Full or half-page illustration of the key action moment
- Text describes the action above/below, image shows the peak moment
- Sound effects in large bold text (BOOM! ZZZT! BZZZ!) overlaid or adjacent
- Used for: fast action, comedic moments, chase scenes, physical comedy
- Multiple smaller spot illustrations can show sequential beats (before → during → after) arranged vertically or diagonally
- **Comic panel grids (3-9 panels) are reserved for special moments only** — when the plot specifically demands it (Sam visualizing a game, dream sequences, rapid-fire comedy beats). Use sparingly, max 2-3 times per book.

### TYPE E: Double-Page Spread
- Both facing pages form one continuous image
- Used 3-5 times per book: opening scene, climactic moments, emotional peaks
- Maximum impact. Kids should want to stare at these for minutes.
- Minimal or no text. Maybe a chapter title overlaid.

### TYPE F: Glitch Page
- Text and images warp, pixelate, fragment during glitch events
- Scan lines cross the page
- Words change size mid-sentence: "Everything was ffffffffffFINE until it W A S N ' T"
- Some letters replaced by pixels or symbols
- Background gets static-noise texture
- Green/blue color cast during digital corruption
- These pages should feel UNCOMFORTABLE — the book itself is glitching

### TYPE G: Data/Blueprint Page
- Maya's invention blueprints with handwritten notes
- Leo's code screens with green-on-black terminal text
- Blip's system diagnostics
- The Gridlord's puzzle fragments
- These break the story format — they feel like found documents
- Hand-drawn arrows, circled notes, coffee stains

### TYPE H: Character Voice Page
- When a character takes over narration briefly
- Page design shifts to match their personality:
  - Maya: blueprint-paper background, gear doodles, handwritten captions
  - Leo: dark screen background, code font, cursor blinking
  - Zara: sketchbook paper background, watercolor splashes, brush-stroke borders
  - Sam: game HUD overlay, health bar at top, score counter, pixel border
  - Blip: minimalist, lots of white space, one LED face expression taking up a quarter of the page, simple text below like a system log

## 3. Image Framing — No Rectangles

Images are never in plain rectangles. They break free.

### Organic Shapes
| Scene Type | Frame Shape | Example |
|-----------|-------------|---------|
| Maya tinkering | Gear-toothed circle | Image clipped into a gear silhouette |
| Leo coding | Monitor/screen shape | Rounded rectangle with a "taskbar" at bottom |
| Zara creating | Paint splatter blob | Irregular splatter outline, paint drips extending below |
| Sam in action | Starburst / explosion | Jagged radiating points around the image |
| Blip scenes | Rounded square | Matches Blip's own body shape |
| Gridlord/digital | Glitch-fragmented | Image split into offset tiles with gaps |
| Nature/outdoor scenes | Leaf or cloud shape | Soft organic edges |
| Emotional moments | Heart-shaped or soft vignette | Faded edges, warm glow |
| Team together | Shield or badge shape | Squad emblem framing |
| Chase/action | Diagonal slash | Image tilted, motion lines extending off-page |

### How It Works With Bonsai Images
Bonsai generates full rectangular images with backgrounds. The backgrounds are PART of the story — a cluttered workshop, a neon city street, a dark basement. We DON'T remove them. Instead:

1. **Full-bleed scenes**: The rectangular image fills the page. Background becomes the page background. No framing needed — the scene IS the page.
2. **Spot illustrations with scene context**: Image is placed at 60-80% size. The background blends into the page via a soft vignette or matching gradient at edges.
3. **Character focus on white**: When showing a character portrait (face, full body reference), prompt Bonsai with "white background" or "simple gradient background." These get the fancy clip-path shapes (gear, starburst, paint splatter).
4. **Comic panels**: Cropped to panel shape. Background is part of the panel story.

## 5. Typography

### Main Body Font
- Warm, friendly, slightly rounded sans-serif
- Must support: Latin + Devanagari + CJK (for multilingual editions)
- Size: 12-14pt at trim size
- Line height: generous (1.5-1.6) — kids need breathing room
- Paragraph: 2-4 sentences max. No block paragraphs.

### Special Typography Treatments

| Element | Treatment |
|---------|-----------|
| Chapter titles | Bold, character-themed color, decorative underline (gears for Maya chapters, pixels for Leo, etc.) |
| Catchphrases | Larger font, character color, bold, with small decorative frame |
| SHOUTING | ALL CAPS, slightly larger, bolder, maybe with impact lines |
| Whispers | Smaller font, lighter weight, italic, maybe in a cloud shape |
| Glitch text | Letters shift vertically, some replaced by ▓▒░, color shifts to purple/blue |
| Code text | Monospace font, green on dark background, like a terminal |
| Sound effects | Comic-style: BOOM!, ZZZT!, BZZZ! — large, bold, outlined, with speed lines |
| Blip's dialogue | Slightly different font, cyan color, small square brackets 【like this】 |

## 6.5 Standardized Character Dialogue

Every character has a FIXED visual treatment for their dialogue. This must be consistent across all books — a child should instantly recognize who's speaking from the styling alone.

### Dialogue Format Rules

**Character Name Label** — Bold, small caps, 11pt, in character's signature color.

**Dialogue Body** — Standard body text in warm charcoal (#2D2D2D) EXCEPT for Blip and the Gridlord (see below).

### Per-Character Standards

**MAYA**
- Name: `MAYA:` in olive green (#556B2F), bold
- Text: Standard body font, normal weight
- Catchphrase: `"I can fix that!"` — larger (16pt), bold, olive green with faint green background pill

**LEO**
- Name: `LEO:` in navy blue (#1B1F3B), bold
- Text: Standard body font, normal weight
- Catchphrase: `"Error 404: [Something] Not Found"` — larger, bold, navy with faint blue background
- When Leo explains tech: slight indent, smaller font (11pt), denser paragraph

**ZARA**
- Name: `ZARA:` in golden yellow (#DAA520), bold
- Text: Standard body font, normal weight
- Catchphrase: `"Hold on, let me DRAW the plan."` — larger, bold, golden with faint yellow background

**SAM**
- Name: `SAM:` in bright red (#E63946), bold
- Text: Standard body font, normal weight — but Sam's lines are often SHORT. One-liners. Punchy.
- Catchphrase: `"CHALLENGE ACCEPTED!"` — larger (16pt), BOLD ALL CAPS, bright red with faint red background
- Sam's excitement is shown through ALL CAPS and exclamation marks, never font tricks

**BLIP** (most distinctive — must be instantly recognizable)
- Name: `BLIP:` in cyan (#00BCD4), monospace font (Fira Code / Consolas), 10pt
- Text: Wrapped in 【corner brackets】, monospace font, cyan color (#00BCD4), 12pt
- Background: faint cyan tint (rgba(0,188,212,0.06)) behind the entire dialogue block
- LED face: Always shown as an emoji-like face in the margin NEXT to Blip's dialogue (◠‿◠, ●_●, ○_○, etc.)
- System messages: `【System: ...】` in dimmer cyan, italic
- Blip NEVER uses contractions in formal mode ("I am" not "I'm"), but picks up contractions as he learns

Example:
```
BLIP:
【I am experiencing what humans call... the creeps.
  Is this normal? I do not enjoy it.】  ○_○
```

**THE GRIDLORD** (even more distinctive)
- Name: `GRIDLORD:` — NO name label. Voice comes from the text itself.
- Text: Purple (#4B0082), slightly larger (14pt), bold italic, with scanline texture behind
- Randomly glitched characters: ▓▒░ interspersed
- No quotation marks — the Gridlord doesn't "speak," the Gridlord IS the text
- Pull-quote style: indented, purple left border, faint purple background

**NARRATOR / BODY TEXT**
- Standard body text, warm charcoal (#2D2D2D)
- Never colored unless it's a pull-quote
- Pull-quotes: italic, bold, bordered left in relevant character color

## 8. Bonsai Prompt Strategy

### Prompt Anatomy for Book Illustrations

Every scene illustration prompt follows this structure:

```
[STYLE TAG] [SCENE DESCRIPTION] [CHARACTER DESCRIPTIONS] [BACKGROUND/SETTING]
[MOOD/LIGHTING] [COMPOSITION] [ART DIRECTION]
```

### Style Tag (Prefix Every Prompt)
```
THE GLITCH SQUAD children's book art, vibrant cel-shaded 2D digital illustration,
bold clean black outlines, flat color fills with minimal soft shading, round friendly
proportions, large expressive eyes, warm golden lighting, bright candy-color palette,
no photorealism no 3D render
```

### When to Use White Background
- Character portraits and reference shots
- Character introduction pages
- Spot illustrations showing a face or expression
- Blueprint/data overlay pages

### When to Use Scene Backgrounds
- Everything else. Action, dialogue scenes, locations, team scenes.
- The background IS part of the story.
- Prompt the background explicitly: "cluttered basement workshop with dusty shelves and old monitors"
- Let the background extend to page edges for full-bleed placement.

### Resolution Targets
| Layout Use | Bonsai Resolution | Notes |
|-----------|-------------------|-------|
| Full-page illustration | 1024×1024 | Main scenes, full bleed |
| Half-page illustration | 1024×1024 | Cropped/clipped in layout |
| Spot illustration | 512×512 | Small vignettes, expressions |
| Double-page spread | 1024×512 (or stitch 2 images) | Panoramic scenes |
| Comic panels | 512×512 each | Individual panels |
| Blip marginalia | 256×256 | Tiny LED face reactions |

## 10. Layout Tool: WeasyPrint Pipeline

### Why HTML/CSS → PDF

1. **Full creative control** — CSS clip-path, transforms, layers, blend modes
2. **Programmatic** — templates for each page type, auto-populated from chapter content
3. **Print-ready output** — WeasyPrint supports CSS Paged Media (@page rules, bleeds, crop marks, page counters)
4. **Multilingual-ready** — HTML natively supports Unicode, RTL, mixed scripts
5. **Free and open source** — Python library, no license costs
6. **Iterative** — change CSS, regenerate, see results in seconds

### Technical Stack
```
Chapter Text (Markdown)
    + Illustration List (JSON manifest)
    + Page Type Assignments
    ↓
HTML/CSS Template Engine (Python/Jinja2)
    ↓
WeasyPrint → Print-Ready PDF (CMYK, 300ppi, bleed)
```

### Key CSS Capabilities We'll Use
- `@page` rules for trim size, margins, bleed
- `clip-path: polygon()` for organic image shapes
- `transform: rotate()` for tilted panels and dynamic layouts
- `mix-blend-mode` for texture overlays
- `background-blend-mode` for decorative layering
- CSS Grid for comic panel layouts
- `shape-outside` for text wrapping around irregular illustrations
- `filter: url()` for SVG glitch effects on text
- `columns` for multi-column text in data pages
- `@media print` for print-specific adjustments

## 11. The Reading Experience

A kid opens this book. Here's what they should feel on every spread:

**Pages 1-2 (Opening)**: Full-bleed cinematic spread of Maple Street at golden hour. Warm. Alive. You can almost hear the sounds. A tiny Blip face in the gutter, peeking at the reader.

**Pages 3-4**: Text on the left with Maya's chapter title (gear decorations). A half-page illustration of Maya's cluttered room on the right — tools everywhere, posters on walls, a half-built gadget on the desk. Text wraps around a small spot illustration of her safety goggles.

**Pages 5-6**: Comic panel page — Maya sneaking into the basement. 6 panels. Sound effects: CREAK... SQUEAK... The last panel is bigger: a dusty NexCorp crate with something glowing inside.

**Pages 7-8**: Full-page illustration — Blip activating. Light pouring from the cube, dust particles floating in the beams, Maya's face illuminated with wonder. No text except the faint glow of Blip's first words: "System initialized."

**Pages 9-10**: GLITCH PAGE. The text itself breaks. Words fragment. Scan lines. The Gridlord's face flickers at the page edges. Kids should feel that something is WRONG with the book itself.

**Pages 11-12**: Back to normal. A sigh of relief. Text page with Blip marginalia showing his scared face (○_○). Small decorative elements: tiny circuit-board pattern in the margins, faintly, as if the glitch left traces.

This rhythm — text, illustration, comic, full-page, GLITCH, relief — repeats with variation throughout the book. The visual pacing mirrors the story pacing.

## 12. Geronimo-Style Markup System

### What It Is

Every page of a Geronimo-style book has **colored, expressive keywords** woven into the body text. These aren't just decorative — they're the book's visual voice. A kid should be able to open to any page and immediately see which character or mood dominates just from the colors.

### Manuscript Markup

In the manuscript (`.md` file), Geronimo keywords use this syntax:

```
~M:text~    → Maya (olive green #556B2F) — building, fixing, inventions
~L:text~    → Leo (navy blue #1B1F3B) — code, data, analysis
~Z:text~    → Zara (gold #DAA520) — art, patterns, beauty
~S:text~    → Sam (crimson red #E63946) — action, games, energy
~B:text~    → Blip (cyan #00BCD4) — robots, processing, learning
~G:text~    → Gridlord/Kira (purple #4B0082) — mystery, glitches, the Grid
~D:text~    → Daadi (brown #8B4513) — wisdom, food, home
```

### Usage Rules

| Rule | Example | Purpose |
|------|---------|---------|
| **Sound effects** | `~S:KRRRZZZT!~` | SFX get larger font, bold, red |
| **Dialogue attribution** | `~B:"I am learning."~` | Character-colored dialogue labels |
| **Emotional narration** | `~M:INCREDIBLE~` | Emphasis words match chapter POV character |
| **Descriptive keywords** | `~G:secrets~` | Mood words match the vibe (purple for mystery) |
| **Catchphrase highlights** | `~M:"I can fix that!"~` | Signature lines in character's color |

### Density Target

- **400-600 colored keywords** per 20K-word book (every ~35 words gets a color pop)
- **Sound effects**: 15-25 per book
- **Dialogue lines**: 60-70% of text should be dialogue (short, punchy, colored)
- **Every page** should have at least 2-3 colored keywords visible

### CSS Implementation

```css
.gk { font-weight: 900; display: inline; }
.gk.maya { color: #556B2F; }
.gk.leo { color: #1B1F3B; }
.gk.zara { color: #DAA520; }
.gk.sam { color: #E63946; font-size: 1.15em; }   /* Sam is LOUDER */
.gk.blip { color: #00BCD4; font-family: monospace; } /* Blip is robotic */
.gk.gridlord { color: #4B0082; font-style: italic; } /* Gridlord is glitchy */
.gk.daadi { color: #8B4513; }
.gk.sfx { font-size: 1.4em; text-transform: uppercase; letter-spacing: 2px; }
```

### AI Generation Prompt

When prompting an AI to write Geronimo-style content:

```
Write in GERONIMO STILTON style:
- Short paragraphs (2-4 sentences max)
- 60-70% dialogue, fast pacing
- Mark colored keywords with ~CHARACTER_CODE:text~
  Maya=~M:, Leo=~L:, Zara=~Z:, Sam=~S:, Blip=~B:, Gridlord=~G:, Daadi=~D:
- Mark sound effects with ~S:BOOM!~ (all caps, onomatopoeia)
- Bold key action words with **bold**
- Every ~35 words should have a colored keyword
- Chapters: 500-1000 words, heavy illustration density
```

### Builder Processing

The `fmt()` function in `build_book.py` converts `~X:text~` to `<span class="gk X">text</span>`. No other changes needed — dialogue handling, catchphrase detection, and page layout all work as before.
